### 현재 기준 메모
- 현재 마감기한은 날짜 + 시간(24시간제)까지 입력 가능하다.
- 현재 반복 요일은 생성 시 여러 개 선택해서 저장할 수 있다.
- 현재 상단 고정 항목은 일반 항목보다 위에 표시된다.
- 현재 드래그 정렬은 같은 고정 그룹 안에서 순서 변경이 가능하다.
- 현재 완료된 항목은 자동으로 리스트 하단으로 이동한다.
- 현재 카테고리는 제목과 탭 색상을 지정해서 생성/삭제할 수 있다.
- 현재 Todo는 카테고리를 지정해서 생성할 수 있고, 리스트에서 카테고리 탭으로 필터링할 수 있다.
- 아직 마감일 기준 정렬과 수정 기능은 구현하지 않았다.
- 추가 기능은 한 번에 여러 개 넣지 않고 하나씩 확장한다.


## 기술스택
- Frontend: React + Vite
- Backend: Node.js + Express
- Database: MongoDB Atlas + Mongoose
- API 통신: Axios
- 환경변수 관리: dotenv
- CORS 처리: cors
- 개발 편의: nodemon
- 스타일링: CSS
- 배포: Vercel
- 버전 관리: Git + GitHub

## 시작 전 단계 정리

### 1. 프로젝트 방향 먼저 확정
- Todo 앱의 핵심 기능과 확장 기능을 구분한다.
- 이번 프로젝트에서는 먼저 기본 CRUD가 되는 MVP를 완성한 뒤, 마감기한, 요일 반복, 상단 고정, 카테고리 정렬 같은 확장 기능을 추가한다.
- 화면보다 기능 완성과 연결 구조 이해를 우선한다.

### 2. 개발 구조 이해하기
- 프론트엔드는 사용자가 보는 화면을 담당한다.
- 백엔드는 요청을 받아 처리하는 서버 역할을 한다.
- 데이터베이스는 Todo 데이터를 저장한다.
- 즉, 사용자가 프론트에서 입력 -> 백엔드가 처리 -> 데이터베이스에 저장 -> 다시 프론트에 결과 표시 흐름으로 이해한다.

### 3. 개발 환경 준비
- Node.js 설치 여부 확인
- Git 설치 여부 확인
- GitHub 계정 준비
- MongoDB Atlas 계정 준비
- Vercel 계정 준비

### 4. 프로젝트 폴더 구조 정하기
- 루트 폴더 아래에 `frontend/`, `backend/` 구조로 나눈다.
- 프론트와 백을 분리해 관리하면 역할이 명확해지고 배포 준비도 쉬워진다.

예상 구조:

```text
project/
├── frontend/
├── backend/
├── README.md
├── before_start.md
└── 기술스택 정리.md
```

### 5. 구현 순서 정하기
- 1단계: 가장 단순한 Todo CRUD 먼저 구현
- 2단계: 프론트와 백엔드 연결
- 3단계: MongoDB 연동
- 4단계: 추가 기능 구현
- 5단계: 테스트
- 6단계: 배포

### 6. 1차 MVP 범위 확정
- Todo 추가
- Todo 목록 조회
- Todo 완료 체크
- Todo 삭제

이 단계에서는 아래 기능은 잠시 보류한다.
- 마감기한
- 요일 반복
- 상단 고정
- 카테고리 정렬
- 마감 날짜순 정렬
- 완료 항목 자동 하단 정렬

이유:
- 처음부터 기능을 많이 넣으면 구조 이해보다 구현 난이도가 급격히 올라간다.
- 기본 CRUD가 먼저 안정적으로 동작해야 이후 기능 확장이 쉬워진다.

### 7. 백엔드 먼저 설계
- Todo 데이터 형태를 먼저 정한다.
- 예시 필드:
  - title
  - completed
  - dueDate
  - repeatDays
  - pinned
  - category
  - createdAt
- 처음 구현은 `title`, `completed`만으로 시작하고 이후 필드를 확장한다.

### 8. API 설계
- `GET /api/todos` : 전체 Todo 조회
- `POST /api/todos` : Todo 추가
- `PUT /api/todos/:id` : 완료 여부 수정 또는 기타 정보 수정
- `DELETE /api/todos/:id` : Todo 삭제

추가 기능 구현 시 확장할 내용:
- 정렬 조건 처리
- 카테고리별 조회
- 상단 고정 우선 정렬

### 9. 프론트엔드 설계
- 입력창
- 추가 버튼
- Todo 목록 영역
- 체크박스
- 삭제 버튼

추가 기능은 기본 기능이 끝난 뒤 아래 순서로 붙인다.
- 마감기한 입력
- 카테고리 선택
- 상단 고정 버튼
- 요일 반복 선택 UI
- 정렬 UI

### 10. 데이터베이스 연동 준비
- MongoDB Atlas 클러스터 생성
- 사용자 계정 생성
- IP 허용 설정
- 연결 문자열 발급
- `.env`에 `MONGODB_URI` 저장

### 11. 테스트 계획 세우기
- Todo 추가가 되는지 확인
- 새로고침 후에도 데이터가 남는지 확인
- 체크 상태가 변경되는지 확인
- 삭제가 정상 동작하는지 확인
- 에러 없이 프론트와 백엔드가 연결되는지 확인

추가 기능 테스트:
- 마감 날짜 정렬
- 카테고리별 정렬
- 완료 항목 하단 이동
- 상단 고정 우선 배치

### 12. 배포 계획 세우기
- GitHub에 프로젝트 업로드
- Vercel에 연결
- 환경변수 등록
- 배포 후 실제 URL에서 동작 확인
